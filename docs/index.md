## Custom Project

### Dataset

The dataset used in this project is `sales.csv`.
It contains sales transaction records with fields such as
order ID, datetime, region ID, product ID,
unit price, quantity, customer ID,
payment method, and online purchase information.

### Data Contract

The data contract requires fields such as order ID,
datetime, region ID, currency code, product ID,
unit price, quantity, customer ID, and payment method.

Messages are considered valid when required fields
are present, quantities are positive,
unit prices are valid, and IDs match the
reference datasets.

Invalid messages are rejected.

### Kafka Messages

The Kafka producer reads sales records from `sales.csv`
and sends them to a Kafka topic one message at a time.

I customized the producer by adding logging that
displays product IDs and prices when messages
are produced.

### Consumer Validation

The consumer validates each message before processing.

Validation checks ensure required fields exist,
quantities are positive, unit prices are numeric
and greater than zero, and IDs match reference data.

Invalid messages are rejected or skipped,
helping maintain data quality.

### Data Engineering and Enrichment

The consumer enriches incoming messages by calculating
derived fields such as subtotal, tax amount, and total.

I added a custom discount calculation that applies
a 10% discount to orders over $100 and created the
derived fields `discount_amount`
and `discounted_total`.

### Streaming Analytics

The consumer maintains running statistics
as messages arrive.

These include total sales, average sale amount,
minimum sale amount, and maximum sale amount.

The statistics update continuously as each message
is processed.

### Experiments

For Phase 4, I modified the data enrichment process
by creating `derived_fields_blessing.py`
and implementing discount calculations
for qualifying orders.

I also modified the consumer to display the new
discount fields in the logs.

For Phase 5, I created custom validation rules in
`data_validation_blessing.py`
and `data_contract_blessing.py`
to ensure that unit prices are numeric and greater
than zero before records are accepted.

### Results

The producer successfully sent sales messages to Kafka,
and the consumer successfully processed all valid messages.

The output CSV contained enriched sales records,
while the logs displayed subtotal, tax, total,
discount amount, discounted total,
and running summary statistics.

### Interpretation

This project demonstrated how streaming data can be
validated, enriched, and analyzed in real time.

I learned how validation protects data quality,
how derived fields add business value,
and how running analytics can provide useful business
intelligence from streaming sales data.
